# Security Policy

## Supported Versions
IBeam is currently under active development. Security fixes are prioritized for the latest published release line.

## Reporting a Vulnerability
Please report security vulnerabilities privately. Do not open a public GitHub issue for undisclosed vulnerabilities.

When reporting, include:
- affected package(s) and version(s),
- clear reproduction details,
- potential impact,
- any proof-of-concept (if safe to share).

## Response Expectations
Target response goals:
- initial acknowledgement: within 3 business days,
- triage and severity assessment: within 7 business days,
- fix timeline communicated after triage.

## Disclosure
After a fix is available, maintainers will coordinate responsible disclosure details and release notes.
