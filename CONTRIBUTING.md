# Contributing to IBeam

Thank you for helping improve IBeam.

## Ways to Contribute
- report bugs and edge cases,
- propose enhancements,
- improve documentation,
- submit pull requests for core framework improvements.

## Workflow
1. Fork the repository.
2. Create a feature branch.
3. Add or update tests for behavior changes.
4. Run `dotnet build IBeam.sln` and `dotnet test IBeam.sln`.
5. Open a pull request with a clear summary and rationale.

## Contribution Standards
- Keep changes focused and reviewable.
- Preserve package boundaries and extension patterns.
- Update README/package docs when behavior changes.
- Prefer additive, backward-compatible changes when practical.

## Licensing of Contributions
By submitting a contribution, you agree that your contribution is licensed under the repository's open-source license for core projects.

## Code of Conduct
Be respectful, constructive, and collaborative.
